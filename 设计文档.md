# 银行业务活动模拟系统 - 设计文档

## 一、问题描述

某银行有4个窗口对外接待客户。从银行开门起每个窗口每时刻只能接待一个客户，因此当客户人数众多时需在每个窗口前排队。对于刚进入银行的客户，如果某个窗口正空闲，则该客户可上前办理业务；如果所有窗口都不空闲，则客户会排在人数最少的队伍后面。

**目标**：编制程序模拟银行的业务活动，并计算一天内客户在银行的平均逗留时间。

## 二、数据结构设计

### 2.1 客户结构（Customer）

```typescript
interface Customer {
  id: number;                    // 客户编号（唯一标识）
  arrivalTime: number;           // 到达时间（分钟）
  serviceTime: number;           // 业务办理所需时间（分钟）
  startTime: number;             // 开始服务时间（分钟）
  endTime: number;               // 离开时间（分钟）
  windowId: number;              // 服务窗口编号（-1表示未分配）
  waitTime: number;              // 等待时间（分钟）
  status: 'waiting' | 'serving' | 'completed';  // 客户当前状态
}
```

**设计说明**：
- `id`：自增编号，用于唯一标识每个客户
- `arrivalTime`：客户到达银行的时刻
- `serviceTime`：客户办理业务所需的时间
- `startTime`：客户开始被服务的时刻
- `endTime`：客户完成业务离开的时刻
- `windowId`：记录客户被分配到哪个窗口
- `waitTime`：客户在队列中等待的时间
- `status`：客户当前所处的状态

### 2.2 窗口结构（Window）

```typescript
interface Window {
  id: number;                    // 窗口编号
  queue: Customer[];             // 等待队列（数组实现的队列）
  currentCustomer: Customer | null;  // 当前正在服务的客户
  totalServed: number;           // 已服务客户总数
  totalWaitTime: number;         // 累计等待时间
  totalServiceTime: number;      // 累计服务时间
  idleTime: number;              // 空闲时间
}
```

**设计说明**：
- `id`：窗口编号，从1开始
- `queue`：使用数组实现的队列，存储等待中的客户
- `currentCustomer`：当前正在办理业务的客户，null表示窗口空闲
- 统计字段用于计算窗口利用率等指标

### 2.3 事件结构（Event）

```typescript
type EventType = 'arrival' | 'departure';

interface Event {
  type: EventType;               // 事件类型
  time: number;                  // 事件发生时间
  customer: Customer;            // 相关客户
  windowId?: number;             // 相关窗口（离开事件需要）
}
```

**设计说明**：
- 采用事件驱动模拟方法
- `arrival`事件：客户到达银行
- `departure`事件：客户完成业务离开

### 2.4 时间线事件（TimelineEvent）

```typescript
interface TimelineEvent {
  time: number;                  // 事件发生时间
  type: 'arrival' | 'start_service' | 'end_service';
  customerId: number;            // 相关客户编号
  windowId?: number;             // 相关窗口
  description: string;           // 事件描述
}
```

**设计说明**：
- 用于记录模拟过程中的所有事件
- 支持动态显示和回放功能

## 三、算法设计

### 3.1 整体流程

```
1. 初始化窗口和事件队列
2. 将所有客户到达事件加入事件队列
3. 按时间顺序处理事件：
   a. 如果是到达事件：
      - 查找空闲窗口
      - 有空闲窗口：直接开始服务
      - 无空闲窗口：加入最短队列
   b. 如果是离开事件：
      - 完成当前客户服务
      - 处理队列中的下一个客户
4. 计算统计数据
```

### 3.2 窗口分配策略

```typescript
// 优先分配空闲窗口
function getIdleWindow(): Window | null {
  return windows.find(w => w.currentCustomer === null) || null;
}

// 获取最短队列窗口
function getShortestQueueWindow(): Window {
  let shortest = windows[0];
  for (const window of windows) {
    if (window.queue.length < shortest.queue.length) {
      shortest = window;
    }
  }
  return shortest;
}

// 分配客户到窗口
function assignCustomerToWindow(customer: Customer): void {
  const idleWindow = getIdleWindow();
  
  if (idleWindow) {
    // 直接服务
    customer.windowId = idleWindow.id;
    customer.startTime = currentTime;
    customer.endTime = currentTime + customer.serviceTime;
    customer.waitTime = 0;
    idleWindow.currentCustomer = customer;
    // 添加离开事件
    events.push({ type: 'departure', time: customer.endTime, customer, windowId: idleWindow.id });
  } else {
    // 加入最短队列
    const shortestWindow = getShortestQueueWindow();
    customer.windowId = shortestWindow.id;
    shortestWindow.queue.push(customer);
  }
}
```

### 3.3 时间计算

- **等待时间** = 开始服务时间 - 到达时间
- **逗留时间** = 离开时间 - 到达时间 = 等待时间 + 服务时间
- **平均逗留时间** = 所有客户逗留时间之和 / 客户总数

### 3.4 窗口利用率计算

```
窗口利用率 = 窗口累计服务时间 / 总模拟时间
```
