## 二、存储结构说明

本系统采用**事件驱动模拟**方式，核心数据结构如下：

### 2.1 客户结构（Customer）
```typescript
interface Customer {
  id: number;                    // 客户编号（唯一标识）
  arrivalTime: number;           // 到达时间（分钟）
  serviceTime: number;           // 业务办理所需时间（分钟）
  startTime: number;             // 开始服务时间（分钟）
  endTime: number;               // 离开时间（分钟）
  windowId: number;              // 服务窗口编号
  waitTime: number;              // 等待时间（分钟）
  status: 'waiting' | 'serving' | 'completed';
}
```

### 2.2 窗口结构（Window）
```typescript
interface Window {
  id: number;                    // 窗口编号
  queue: Customer[];             // 等待队列（数组实现）
  currentCustomer: Customer | null;
  totalServed: number;           // 已服务客户总数
  totalWaitTime: number;         // 累计等待时间
  totalServiceTime: number;      // 累计服务时间
  idleTime: number;              // 空闲时间
}
```

**说明**：每个窗口的等待队列采用**数组实现的队列结构**，支持 FIFO（先进先出）操作。

### 2.3 事件结构（Event）
```typescript
interface Event {
  type: 'arrival' | 'departure';
  time: number;                  // 事件发生时间
  customer: Customer;
  windowId?: number;
}
```

**说明**：事件表存储所有到达和离开事件，**按时间顺序处理**。每次处理事件前，事件表会重新排序以确保时间有序性。

### 2.4 时间线事件（TimelineEvent）
```typescript
interface TimelineEvent {
  time: number;
  type: 'arrival' | 'start_service' | 'end_service';
  customerId: number;
  windowId?: number;
  description: string;
}
```

**说明**：用于记录模拟过程中的关键事件，支持动态展示和回放。

## 三、核心算法说明

### 3.1 最短队列优先（SQF）算法

这是题目要求的基础算法：

```typescript
function assignCustomerToWindow(customer: Customer): void {
  // 1. 检查是否有空闲窗口
  const idleWindow = this.windows.find(w => w.currentCustomer === null);
  
  if (idleWindow) {
    // 空闲窗口直接服务
    customer.windowId = idleWindow.id;
    customer.startTime = customer.arrivalTime;
    customer.status = 'serving';
    idleWindow.currentCustomer = customer;
  } else {
    // 2. 所有窗口都忙，找队列最短的窗口
    const shortestQueue = this.windows.reduce((min, w) => 
      w.queue.length < min.queue.length ? w : min
    );
    
    customer.windowId = shortestQueue.id;
    customer.status = 'waiting';
    shortestQueue.queue.push(customer);
  }
}
```

**算法特点**：
- 优先使用空闲窗口
- 如果所有窗口都忙，选择队列最短的窗口排队
- 时间复杂度：O(n)，n 为窗口数量

## 八、答辩准备

### 可能被问到的问题及回答

**Q1：为什么选择事件驱动模拟？**
A：事件驱动模拟能够准确模拟离散事件系统。银行业务是典型的离散事件系统，客户到达和离开都是离散的时间点。通过维护事件表并按时间顺序处理，可以精确计算每个客户的等待时间和逗留时间。

**Q2：平均逗留时间是如何计算的？**
A：逗留时间 = 离开时间 - 到达时间 = 等待时间 + 服务时间。系统记录每个客户的到达时间和离开时间，最后对所有完成服务的客户求平均值。

**Q3：如何保证事件按时间顺序处理？**
A：在每次处理事件前，都会对事件表进行排序（`events.sort((a, b) => a.time - b.time)`），确保总是处理时间最早的事件。

**Q4：窗口队列为什么用数组实现？**
A：数组可以高效实现队列的 FIFO 操作。使用 `push()` 在队尾添加客户，使用 `shift()` 从队头取出客户，符合队列的先进先出特性。

**Q5：三种算法的性能差异是什么？**
A：
- SQF（最短队列）：简单直观，但可能导致某些窗口负载不均
- RR（轮询）：负载均衡，但不考虑实际队列长度
- LEW（最短预期等待）：综合考虑队列长度和当前服务时间，通常性能最优
